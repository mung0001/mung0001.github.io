    Docker 기본 명령어

    • <Docker container run> <Docker Imagename> <명령어>
      • <Docker container run : 컨테이너 작성 및 실핼
      • <Docker Imagename : 바탕이 되는 Docker 이미지
      • <명령어> : 컨테이너 안에서 실행할 명령어

    $ docker version
# docker 버전확인

$ docker system info
# docker 실행환경확인

$ docekr system df
# docekr 디스크 사용량 확인

$ docker ps
# 서버의 상태 확인

$ docker pull <name>
$ name Image 다운로드

$ docker images
# 이미지 확인

$ docker container run --name <name> -d -p <host port num>:<container port num> nginx
# 서버 기동

$ docker stats <name>
# name의 정보를 출력

$ docker stop <name>
# name의 컨테이너를 정지

$ docker kill <name>
# name 컨테이너를 강제정지

$ docker unpause <name>
# 정지된 name 컨테이너를 재개

$ docker start <name>
# name의 컨테이너를 시작

$ docker container rm <옵션> <컨테이너 식별자>
# -f : 실행 중인 컨테이너 강제 삭제 / -v : 할당한 볼륨을 삭제
# 해당 옵션으로 식별자의 컨테이너를 삭제한다 

$ docker image inspect <name>
# name의 이미지의 상세정보 출력 ( CPU, OS etc...)

$ docker image tag <name> <new name:new version>
# name의 이미지의 이미지 및 태그로 수정

$ docker image rm <옵션> <이미지명:태그>
# 이미지명:태그의 이미지 삭제
# -f : 강제삭제 / --no-prune : 중간이미지를 삭제하지 않음

$ docker image prune <옵션>
# 사용하지 않는 이미지 삭제
# -all, -a : 사용하지 않는 이미지를 모두 삭제 / -f : 강제로 삭제

$ docker image push <이미지명:태그명>
# 이미지 upload

$ docker logs -t <name>
# name 이름의 컨테이너 로그를 출력

$ docker network ls <옵션>
# -f : 필터 / --no-trunc : 상세정보 출력 / -q : network ID만 표시
# network 종류 출력

$ docker network create -d <network type> <name>
# network type의 name network 생성

$ docker docker container stats <컨테이너 식별자>
# 해당 컨테이너의 상태를 출력

     

    Docker Images

    • Docker 이미지 형식

      • 이미지명 : 태그명

    • Docker Store : https://store.docker.com $
    $ docker image pull <옵션> <이미지명:태그명>
# 선택한 옵션 값으리 특정 이미지의명의 태그 이미지를 다운로드
# -a 옵션은 모든 태그의 취득이 가능 ( 태그지정은 불가 )

     

    • 이미지 출력
    $ docker image ls <옵션> <리포지토리명>

    • 옵션

      • -all, -a : 모든 이미지 표시

      • –digests : 다이제스트 표시 유무

      • –no–trunc : 결과를 모두 표시

      • –quiet, -q : Docker 이미지 ID만 표시

     

    • 이미지 검색
    $ docker search <옵션> <검색 키워드>
# 옵션의 값으로 검색 키워드를 서치

    • 옵션

      • –no-trunc : 결과를 모두 표시

      • –limit : n건의 검색 결과를 표시

      • –filter=stars=n : 즐겨찾기의 수(n 이상)을 지정

     

    • 이미지 위장, 변조 제거

    $ export DOCKER_CONTENT_TRUST=1
# 상단의 명령어 입력시 이미지 검증작업을 수행 ( 0은 무효화 )

     

    컨테이너 생성 및 시작

    • Docker Container run
    $ docker container run <옵션> <이미지명:태그명> <인수>

    • 옵션

      • –attach, -a : 표준 입력, 출력, 오류 출력을 어태치

      • –cidfile 컨테이너 ID 파일오 출력

      • –detach -d : 컨테이너를 생성하고 백그라운드에서 실행

      • –interctive, -l : 컨테이너의 표준 입력을 연다

      • –tty, -t : 단말기 디바이스를 사용

      • –user, -u : 사용자명 지정

      • -restart=num : 명령의 실행 결과에 따라 재시작을 진행

      • –rm : 명령 실행 완료 후 자동으로 삭제

     

    • 자원을 지정하여 컨테이너 생성 및 실행 ( docker container run )
    $ docker container run <자원 옵션> <이미지명:태그명> <인수>

    • 옵션

      • –cpu-shares, -c : CPU의 사용비율

      • –memory, -m : Memory 사용비율 ( 단위 b, k, m, g )

      • –volume=[호스트의 디렉토리:[컨테이너의 디렉토리]], -v : 호스트와 컨테이너의 디렉토리 공유

     

    • Docker Container 환경변수
    $ docker container run <환경설정 옵션> <이미지명:태그명> <인수>

    • 옵션

      • –env=<환경변수>, -e : 환경변수를 설정

      • –env-file=<파일명> : 환경변수를 파일로부터 설정

      • –read-only=<true | false> : 컨테이너의 파일 시스템을 읽기전용으로 설정

      • –workdir=, -w : 컨테이너의 작업 디렉토리를 지정

      • –user=<사용자명>, -u : 사용자명 또는 UID를 지정

     

    • Docker container network run

      • –add-host= : 컨테이너의 /etc/hosts에 호스트명과 IP주소를 정의

      • –dns= : 컨테이너용 DNS 서버의 IP 주소 지정

      • –expose : 지정한 범위의 포트 번호를 할당

      • –mac-address= : 컨테이너의 MAC 주소를 지정

      • –net=[bridge | none | container:<name | id> | host | NETWORK] : 컨테이너의 네트워크를 지정

      • –hostname, -h : hostname 지정

      • –publish, -p : 호스트와 컨테이너 포트 매핑

      • –publish-all, -P : 호스트의 임의의 포트를 컨테이너에 할당

     

     

    Docker Hub

    $ docker login <옵션> <서버>
# 해당 서버에 특정 옵션으로 login
# -p : pw / -u : 사용자명

     

    • Docker 출력 형식 지정
    $ docker container ls -a "{{.Names}}: {{.Status}}"...

    • 옵션

      • .ID : 컨테이너 ID

      • .Image : 이미지 ID

      • .Command : 실행명령

      • .CreatedAt : 컨테이너가 작성된 시간

      • .RunningFor : 컨테이너 가동시간

      • .Ports : 공개포트

      • .Status : 컨테이너 상태

      • .Size : 컨테이너 디스크 크기

      • .Names : 컨테이너명

      • .Mounts : 불륨 마운트

      • .Networks : 네트워크명
