    Docker Install

     

    Docker Insatll ( Ubuntu )

    $ sudo apt-get update
$ sudo apt-get updgrade
# apt 업데이트

$ sudo apt-get install -y \
    apt-transport-https \
    ca-certificates \
    curl \
    software-properties-common
# 패키지 설치

$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
# Docker의 공식 GPG 키를 추가

$ sudo apt-key fingerprint 0EBFCD88
pub   rsa4096 2017-02-22 [SCEA]
      9DC8 5822 9FC7 DD38 854A  E2D8 8D81 803C 0EBF CD88
uid           [ unknown] Docker Release (CE deb) <docker@docker.com>
sub   rsa4096 2017-02-22 [S]
#GPG Key 확인

$ sudo add-apt-repository \
    "deb [arch=amd64] https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) \
    stable"
# 레포지터리 등록


$ sudo apt-get install -y docker-ce
# docker 설치
    • Cloud 혹은 다른 환경에서의 설치 : https://docs.doocker.com/installation/#installation

     

    Docker 기본 명령어

    • <Docker container run> <Docker Imagename> <명령어>
      • <Docker container run : 컨테이너 작성 및 실핼
      • <Docker Imagename : 바탕이 되는 Docker 이미지
      • <명령어> : 컨테이너 안에서 실행할 명령어

    $ docker version
# docker 버전확인

$ docker system info
# docker 실행환경확인

$ docekr system df
# docekr 디스크 사용량 확인

$ docker ps
# 서버의 상태 확인

$ docker pull <name>
$ name Image 다운로드

$ docker images
# 이미지 확인

$ docker container run --name <name> -d -p <host port num>:< container port num> nginx
# 서버 기동

$ docker stats <name>
# name의 정보를 출력

$ docker stop <name>
# name의 컨테이너를 정지

$ docker start <name>
# name의 컨테이너를 시작

     

    Docker Images

    • Docker 이미지 형식

      • 이미지명 : 태그명

    • Docker Store : https://store.docker.com $
    $ docker image pull <옵션> <이미지명:태그명>
# 선택한 옵션 값으리 특정 이미지의명의 태그 이미지를 다운로드
# -a 옵션은 모든 태그의 취득이 가능 ( 태그지정은 불가 )

     

    $ docker image ls <옵션> <리포지토리명>
