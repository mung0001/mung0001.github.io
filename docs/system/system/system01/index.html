    Operation System Structures

     

    컴퓨터의 정보 처리방식

    • 정보의 최소 단위 " bit(binary digit)

    • 정보의 처리 : 정보의 상태 변환 ( 0 -> 1, 1 -> 0 )

    • 부울 대수 (Boolean Algebra) : NOT, AND, OR

    • 논리 게이트 : NOT, AND, OR, XOR, NAND, NOR

    • 논리회로 : IC, LSL, VLSI, ULSI, SoC…

    • 정보의 저랑 플립-플롭, 데이터 버스, RF 등

    • 덧셈은 가산기, 뺄셈은 2의 보수로 표현

      • 이와 동일하게 곰셈과 나눗셈도 덧셀, 뺄샘을 반복
      • 즉, NOT, AND, OR 게이트만으로 모든 계산이 가능(NAND)

    • 실수연산은 부동 소수점 표현법

    • 함수는 GOTO

     

    Program

    • Program이란 Instructions로 명령어의 집합을 의미

    • OS또한 항상 작동하는 Program이라 할 수 있으며, H/W를 따로 관리하지 않을 수 있도록 H/W와 application programs 사이에서 가교역할을 수행 (Processes, resources, user interface 등을 관리)

     

    Process Concept

    • Process 생명주기
      • New : Process가 생성될 떄
      • Running : Process가 현재 실행 중인 상태
      • Waiting : 실행중인 프로세스가 다른 Process에게 점유를 빼앗겨서 현재 대기 중인 상태
      • Ready : Waiting 상태에서 다시 Running 상태로 진입 전에 대기상태
      • Terminated : Process가 작업을 마쳐 종료된 상태

     

    PCB (Process Control Block) or TCB (Task Control Block)

    • Process 우선순의 정리를 위해 관련 정보를 가지고 있는 블럭

     

    용어정리

    • bootstrap : computer power-on시 가장 첫번째로 Memory상에 올라가며, OS를 memory에 Loading 시킴

    • Interrupts : I/O device에서 특정 행동시 CPU에 시그널을 과정 (ex: 키보드 입력시 CPU에서 이를 인식)

    • computer에 내릴 수 있는 명령을 instruction Set이라 함

    • 속도 순서

      • Register > Cache > Main memory > SSD (soild-state disk) -> hard disk -> optical disk -> magnetic tapes

    • CPU : 중앙 처리 장치

      • Processor : CPU에 포함되어 있는 물리적인 논리회로
      • Core : CPU의 계산 유닛
      • Thread : CPU로 부터 할당받은 자원을 이용하는 실행단위 (CPU = 몸통, Thread = 손)
      • Multicore, Multiprocessor : SMP(Symmetric Multiprocessing) 단일 CPU가 아닌 여러 CPU가 혼용되어 있는 것
      • Multi-core design : 하나의 CPU의 여러 Core가 존재하는 것

    • BUS : OS(그 중 Memory)와 특정 기기(disk, mouse 등)와 연결되어있는 것을 의미

    • Multiprogramming : 여러 프로세스를 올리는 것 (multiprocessing)

      • CPU scheduling : 여러 작업에 대해 우선순위를 정하는 것

    • System call : System(OS)에 특정 행동을 요청하는 것
