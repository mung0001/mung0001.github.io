    DNS 서버 #

    #

    • DNS(Domain Name Server)는 TCP/IP 네트워크에서 이름 해석 서비스를 제공하는 서버 애플리케이션으로 이름 해석이란 호스트 이름과 도메인 이름을 IP 주소로 변환하는, 혹은 반대로 IP 주소에서 호스트 이름이나 도메인 이름으로 변환하는 작업이다.

    • DNS 서버는 자신의 도메인 정보를 관리하고 외부 DNS 서버의 문의에 대응하는 기능을 수행하며, DNS 서버가 관리하는 도메인의 범위를 영역(Zoen)이라 한다.

    #

    Domain 이름의 기술 형식 #

    #

    • 도메인 이름 : 서브도메인 이름 또는 공백

    • 서브 도메인 이름 : 레이블 또는 서브도메인 이름.레이블

    • 레이블 : 문자, 숫자, 하이픈, 스티링, 문자 및 숫자

    • 문자 숫자 하이픈 스트링 :

    • 문자 숫자 하이픈

    • 또는

    • 문자 숫자 하이픈 문자 숫자 하이픈 스트링

    • 문자 숫자 하이픈 : 문자 숫자 또는 -(하이픈)

    • 문자 숫자 : 문자 또는 숫자

    • 문자 : 52개의 A에서 Z사이의 대소 영문자

    • 숫자 : 0~9까지의 숫자들

    #

     

    • 인터넷상에서 IP 패킷의 송수신자는 IP주소로 식별되지만, 애플리케이션인 메일이나 웹에서는 일반적으로 호스트 이름으로 송수신자를 식별한다.

    • 웹 브라우저의 경우 “http://www.example.com” 등으로 서버 이름을 접근하며, 반면에 메일이나 웹 등 데이터를 송신하는 시스템의 IP모듈은 애플리케이션이 지정한 상대편 서버 이름을 이해할 수 없으므로 지정된 서버 이름을 IP 주소로 변환할 필요가 있으며, 이를 DNS의 이름해석이라 한다.

    • DNS 컴포넌트는 이름 해석 서비스를 제공하는 DNS 서버에서 데몬 프로세스로 동작하는 네임 서버와 그 네임 서버가 참조하는 DNS 영역 파일이 존재하며, 반면 이름 해석을 의뢰하는 DNS 클라이언트에는 리졸버가 존재한다.

    • DNS의 구성

      • 도메인 이름 공간과 리소스 레코드 : 도메인 구성과 리소스를 정의하는 DNS 영역 파일

      • 네임 서버 : 서버의 데몬 프로세스로 동작하는 네임 서버 프로그램

      • 리졸버 : 네임 서버에 문의하는 유저 루틴으로 리졸버 구성 파일을 사용

    Mail Server #

    #

    • 메일서버는 우리가 사용하는 메일서버와 동일하며 여기서는 sendmail을 이용하여 서버를 구축한다.

    • 이 때 sendmail의 구성 파일인 sendmail.cf는 WIDE 프로젝트의 CF 패키지를 이용해서 작성한다.

    #

     

    메일 서버의 구조 #

    #

    • 메일 서버(smtp/pop3) 설정과 구체적인 메일 송수신 구조를 설정 파일과 연계하여 설명할 수 있으며, 이 글에서 사용하는 메일 서버 소프트웨어는 sendmail이다.

    • 메일 클라이언트 소프트웨어(smtp 클라이언트)에서 작성, 송신된 메일을 smtp 서버(메일 서버)는 일단 임시 보관 영역(메일 송신 큐)에 메일을 넣은 다음 전송 처리를 진행한다.

    • 메일주소를 보고 서버에 등록된 주소로 보내는 메일이면 계정별 메일 보관함으로 전송하며, 외부로 보내는 메일이면 해당 주소 도메인의 smtp 서버로 메일을 전송한다.
