    UNIX/ Linux 기본 명령어와 개념 #

    #

    Redirection & Pipes #

    # 

    $ <File Descriptor> <PATH> <Redirection>
$ <File Descriptor> <PATH> <Redirection> <Pipe> <File Descriptor> ...
ex) $ cat /log/var/message | grep ERR

# cat = <File Descripotr>

# /log/var/message = <PATH>

# | = <Redirection>

    #

    • File Descriptor

      • Process가 File이나 Device를 access 하기 위해 사용

      • Standard File Descriptor

      • stdin(standard input) : 표준 입력 (예시 : 키보드)

      • stdout(standard output) : 표준 출력 (예시 : 모니터)

      • stderr(standard error) : 표준 에러 출력 (예시 : 에러 메시지)

    #

     

    • PATH

      • 파일의 경로

      • 절대경로 및 상대경로로 구분

    #

     

    • Redirction

      • 출력 재지정

      • 입력 재지정

      • >, » (stdout을 파일로 저장 또는 추가)

      • < (파일을 stdin으로 전달)

    #

     

    • Pipe (|)

      • 앞의 명령어의 샐행 결과가 뒤의 명령어에 입력되도록 입출력을 연결하는 역할을 수행

      • Process 연결 (stdout을 stdin으로 전달)

      • Process 간의 데이터 흐름은 자동으로 조절되어짐

    #

     

    • 예시
    $ ls -al > lsoutput.txt
# ls -al의 출력을 lsoutput.txt에 입력
# ls -al = stdout
# > = Redirction
# lsoutput.txt = PATH & stdin

$ cat lsoutput.txt
drwxr-xr-x 2 root root 4096 Aug 23 06:49 .
drwx------ 6 root root 4096 Aug 23 06:49 ..
-rw-r--r-- 1 root root    0 Aug 23 06:49 lsoutput.txt
# lsoutput.txt의 내용을 출력
# cat = stdout
# lsoutput.txt = PATH

$ more < lsoutput.txt
drwxr-xr-x 2 root root 4096 Aug 23 06:49 .
drwx------ 6 root root 4096 Aug 23 06:49 ..
-rw-r--r-- 1 root root    0 Aug 23 06:49 lsoutput.txt
# lsoutput.txt의 내용을 more의 출력
# more = stdout
# < = Redirction
# lsoutput.txt = PATH

$ ps >> lsoutput.txt
$ cat lsoutput.txt
drwxr-xr-x 2 root root 4096 Aug 23 06:49 .
drwx------ 6 root root 4096 Aug 23 06:49 ..
-rw-r--r-- 1 root root    0 Aug 23 06:49 lsoutput.txt
    PID TTY          TIME CMD

   1538 pts/0    00:00:00 su
   1539 pts/0    00:00:00 bash
   2195 pts/0    00:00:00 ps
# ps의 출력 내용을 lsoutput.txt의 이어 쓰기
# ps = stdout
# >> = Redirction
# lsoutput.txt = PATH & stdin


$ ps > lsoutput.txt
$ cat > lsoutput.txt
    PID TTY          TIME CMD
   1531 pts/0    00:00:00 sudo
   1538 pts/0    00:00:00 su
   1539 pts/0    00:00:00 bash

# 여기서 > 는 덮어쓰기 >> 는 이어쓰기를 의미
# ps = stdout
# > = Redirction
# lsoutput.txt = PATH & stdin

    Shell Programming의 방법 #

    #

    • Shell Programming을 사용하기 위해서는 두 가지 방법이 존재

    #

    Line command # 

    $ for file in *
> do
> if grep -l ps $file
> then
> more $file
> fi
> done
lsoutput.txt
    PID TTY          TIME CMD
   1531 pts/0    00:00:00 sudo
   1538 pts/0    00:00:00 su
   1539 pts/0    00:00:00 bash
   2335 pts/0    00:00:00 ps
    • Line command는 CLI환경에서 특정 파일을 통해서가 아닌 직접 함수와 변수를 코딩함으로서 Programming을 수행하는 방법

    #

     

    Script # 

    $ vi Script.sh
# !/bin/bash

for file in *
do
    if grep -l ps $file
    then
        echo $file
    fi
done
exit 0
$ chmod 700 ./Script.sh
$ ./Script.sh
lsoutput.txt
    PID TTY          TIME CMD
   1531 pts/0    00:00:00 sudo
   1538 pts/0    00:00:00 su
   1539 pts/0    00:00:00 bash
   2335 pts/0    00:00:00 ps
    • Script는 특정 파일 내부에 함수와 변수의 코드함으로서 이를 실행시켜 Programing을 수행하는 방법
