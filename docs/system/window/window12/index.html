    Windows Web Server IIS, FTP Server

    IIS(Internet Information Services)

    • WIndows Server 2016에 내장되어 있는 웹 서버 및 FTP 서버는 IIS라는 이름으로제공됩니다.

     

    IIS 실습

    • IIS 설치를 위해 FIRST Server에서 서버매니저 > 관리 > 역할 및 기능 추가를 선택합니다.

     

    • 서버역할에서 Web Server IIS를 선택 후 킷을 포함하여 설치를 진행합니다.

     

    • 역할 서비스 에서 Application Development > CGI 및 FTP Server를 설치하기 위해 체크 후, 설치합니다.

     

    • 설치가 완료되면 방화벽에 허용되었는 지와 서비스가 실행중인지를 확인합니다.

    • IIS는 World Wide Web, FTP는 Microsoft FTP Service입니다.

     

    • 설치가 완료되면 IIS를 설정을 위해 서버 매니저 > 도구 > IIS 관리자를 선택합니다.

     

    • 현재 FTP 서버가 등록되어 있지 않으므로 사이트 > FTP 사이트 추가를 선택합니다.

     

    • 경로를 지정한 후 저장합니다.

     

    • SSL은 사용하지 않을 것이므로 설정하지 않습니다.

     

    • 모두가 접속할 수 있도록 익명, 모든 사용자, 읽기를 설정합니다.

     

    • WINCLIENT에서 192.168.10.11로 접속하면 IIS를 확인하실 수 있습니다.

     

    • 위와 동일하게 FTP 서버 접속을 위해 WINCLIENT에서 ftp 192.168.10.11로 접속합니다.

    • 접속 후에는 anonymous(익명사용자) 및 패스워드는 그냥 엔터를 입력하여 접속합니다.

    • 접속이 확인할 수 있습니다.

     

    • 기본적으로 웹 사이트는 IIS 괸라자 > 사이트 Default Web Site > 기본 문에서 확인 및 설정이 가능합니다.

    • 이제 본격적인 IIS 활용을 진행해보도록 하겠습니다.

     

    IIS를 활용한 웹 사이트 구축

    • IIS를 활용하면 웹 사이트를 구축할 수 있습니다.

    • 웹 사이트 구축을 위해 필요한 기본적인 3요소

      • 웹 서버 소프트웨어 : IIS, Apache 등

      • 웹 프로그래밍 언어 : ASP.NET, ASP, JSP, PHP 등

      • 데이터베이스 툴 : SQL Sefver, Oracle, MySQL 등

    • WPI(Web Platform Installer) : Microsoft에서 제공하는 웹 플랫폼

      • Internet Information Services(IIS), SQL Server Express, Net Framework, 오픈 소스인 PHP와 mySQL과 같은 소프트웨어 제공

     

    XpressEngine를 통한 웹 페이지 실습

    • 먼저 FIRST Server에서 XpressEngine를 설치합니다.

    • 설치 후, 창을 종류 후, IIS 관리자 > 사이트 > Default Web Site > 웹 플랫폼 관리자를 선택합니다.

     

    • 웹 서버를 실행하면 PHP를 검색하여 PHP 5.4.9를 설치합니다.

    • PHP를 선택 후, 설치할 항목에서 IIS용 PHP 관리자를 제거합니다. (호환성 문제)

     

    • 설정이 왼료되면 MySQl를 검색 후, MySQl 5.5버전을 추가 후 설치합니다.

     

    • 암호를 입력하고 설치를 진행합니다.

     

    • 설치가 완료되었습니다.

     

    • 설치가 완료되면 cmd를 시행하여 mysql -u root -p 실행하여 암호를 입력합니다.
    mysql> create database xeDB;
# DB라는 데이터베이스 만듭니다.

mysql> grant all privileges on *.* to xebUser@'127.0.0.1' identified by '1234';
# WebUser라는 계정을 1234의 PW를 가지고 생성하고 자신의 모든 DB를 사용할 수 있는 권한을 부여합니다.

mysql> exit
# MySQL 쿼리를 종료합니다.

     

    • DB 설정이 완료되면 Xe를 다운받습니다.

    • IIS의 기본 폴더인 C:\inetpub\wwwroot\에 붙여넣습니다.

     

    • 이어서 게시판은 모든 사용자가 사용해야 하기 때문에 권한에 대한 설정을 진행합니다.

    • IUSR을 생성 후, 모든 권한을 부여합니다.

     

    • WINCLIENT에서 192.168.10.11/xe로 접속하여 설정을 진행합니다.

     

    • MySQL을 지정합니다.

     

    • 서버에서 설정했던 값들을 기입합니다.

     

    • 어드민 계정을 생성합니다.

     

    • 이제 다시 192.168.10.11/xe로 접속하면 다음과 같은 페이지를 확인하실 수 있습니다.

     

    • 게시판 생성을 위해 하단의 그림과 같이해 사이트 메뉴 편집 > 메뉴 추가 > 게시판을 추가합니다.

     

    • 로그아웃 후, 게스트의 궈한으로 Board에 접속하여 글을 작성합니다.

     

    • 게시판을 확인하실 수 있습니다.

     

    웹 하드 구축

    • 웹 하드는 인터넷 상에서 디스크 공간을 사용하는 기능을 제공하는 사이트를 의미합니다.

     

    웹 사이트를 통한 웹 하드 구축

    • FIRST 서버에 Pydio를 설치합니다.

    • Xe와 동일하게 사용권한을 수정 후, 폴더명을 접근을 쉽게 WebHard로 바꾸어줍니다.

     

    • C:\Window\Temp폴더 또한 권한을 설정해줍니다.

     

    • 설정이 완료되면 WINCLINT에서 192.168.10.11/WebHard에 접속합니다.

    • 접속이 완료되면 에러와 오류를 무시하고 설치를 진행합니다.

     

    • 데이터베이스는 새로 생성해도 되지만 기존의 XeDB와 유저르 사용해보겠습니다.

     

    • 설정이 완료되면 설치를 진행합니다.

     

    • 설치가 완료되면 `192.168.10.11/WebHard로 접소하면 접속이 완료됩니다.

     

    • 새로운 계정생성을 위해 우측의 메뉴에서 설정을 선택합니다.

     

    • People > 새 사용자를 선택하여 사용자를 생성합니다.

    • 생성된 사용자를 선택하면 사용자를 설정할 수 있습니다.

     

    • 재접속 후, 새로 생성된 유저로 접속하여 업로드 및 생성을 할 수 있습니다.

    • 이와 같이 개인적으로 혹은 공유하는 폴더를 웹을 통해 관리할 수도 있습니다.
