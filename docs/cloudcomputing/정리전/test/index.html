    awscli awscli 는 AWS Command Line Interface 의 약어입니다. 설치하면 이후 s3 관련 작업을 terminal command 로 할 수 있습니다.

    설치는 pip install 로 가능합니다. Python 2.6x 혹은 Python 2.7x 이상이 필요하였는데, 저는 Python 3.x 만 이용하므로 버전은 자세히 보지 않았습니다.

    pip install awscli 설치 후 help 명령어를 입력하여 awscli 가 설치되었는지 확인도 해봅니다.

    aws help aws cli 를 이용하기 전에 앞서 만든 AWS IAM 사용자를 추가해야 합니다. aws configure 를 실행시킵니다.

    aws configure 네 개의 항목이 나오는데, access key id 와 secret access key 는 앞서 다운로드 받은 csv 파일 안에 있습니다. region name 은 default region 을 설정하는 것으로, 아시아 (서울)은 ap-northeast-2 입니다. 마지막은 enter 를 눌러 넘어갑니다.

    AWS Access Key ID [None]: AWS Secret Access Key [None]: Default region name [None]: Default output format [None]: 제대로 configure 가 실행 되었는지 확인합니다. 내 계정의 버킷 리스트나 한 버킷 내 파일 리스트를 살펴볼 수 있습니다.

    내 계정의 버킷 리스트

    aws s3 ls

    내 버킷 내 파일 리스트

    aws s3 ls s3://mybucket/ 파일 복사는 aws s3 cp 이후 source, destination 순으로 입력합니다. S3 의 주소는 s3:// 로 시작합니다.

    aws s3 cp localfile s3://[BUCKETNAME]/[FILENAME] 여러 개의 파일이나 폴더를 recursive 하게 복사할 때에는 sync 를 이용할 수 있습니다.

    aws s3 sync SOURCE_DIR s3://DEST_BUCKET/ 업로드 하는 파일을 모두가 읽을 수 있도록 권한을 설정하려면 –acl public-read 옵션을 추가합니다.

    aws s3 sync SOURCE s3://DESTINATION –acl public-read 데이터 백업을 위해서라면 Storage class 를 Standard 로 할 필요는 없습니다. Standard IA 는 자주 접속하지는 않지만, 중요한 데이터를 저장하는 storage class 입니다. 이를 이용하려면 다음처럼 –storage-class 옵션을 부여합니다. 더 많은 cp 관련 옵션은 여기에 있습니다.

    aws s3 sync SOURCE_DIR s3://DEST_BUCKET/ –storage-class STANDARD_IA

    시험 정리

    • S3와 S3-IA는 내구성은 동일하지만 가용성은 S3가 9가 큼 ( S3의 가용성은 99.99이며 S3-IA의 가용성은 99.9 )

    • 데이터가 로컬에 존재하는 경우 스토리지 게이트웨이가 최상의 옵션을 제공

    • 개별 객체의 최대크기는 5TB, AWS는 100MB보다 큰 모든 객체에 멀티 파트 업로드를 사용하는 것이 용이

    • S3의 최종 일관성은 삭제 및 기존 개체를 덮어 씌우는 것

    • S3의 저장할 수 있는 가장 작은 파일은 0byte이지만, 모든 객체가 128KB 이상인 것처럼 요금이 청구되어짐

    • S3를 프로그래밍 방식으로 업데이트 하면 https://[ 리전 주소 ] [ 버킷 주소 ]/ 호스팅 중일 경우 반대

    • S3 One Zone-IA의 가용성은 99.5 S3-IA는 99.9% S3는 99.99%

    • S3와 IAM은 전역 적으로 표시 됨 ( 리전 구분 X )

    • EBS는 객체 기반 스토리지 및 다양한 SSD 및 자기 옵션으로 제공

    • AMI는 지역변수

    • 인스턴스가 종료되면 EBS의 루트 볼륨도 종료되어짐

    • EBS는 증분백업

    • EBS 볼륨은 하나의 한 인스턴스에만 연결이 가능

    • 메타 데이턴느 인증서와 태그를 통해서 추가 됨

    • 개별 인스턴스는 특정 가용성 존 B에서 프로비저닝 됨

    • 여러 가용영역에 배치 가능한 것은 스프레드배치 그룹

    • VPN과 연결을 구축하기 위해 사용되는 서비스는 고객 게이트웨이

    • 일반적으로 VPn 연결은 중복성을 위해 두 개의 서로 다른 터널을 사용

    • 스토리지 나오면 걍 스토리지 게이트웨이

    • NFS 기반 백업 시스템은 파일 게이트웨이

    • 테이프 백업 시스템이 나오면 테이프 게이트웨이 ( Amazon Glacier )

    • 저장된 볼륨 게이트웨이는 S3로 비동기적으로 백업, 네트워크 대기 시간이 최소화

    • 캐시 된 볼륨 게이트웨이는 전체 데이터 세트를 S3에 유지하면서 가장 일반적으로 엑세스되는 데이터를 로컬에 저장

    • 전체 데이터 세트가 필요한 경우 저장된 볼륨 게이트웨이, 특정시간의 우선 순위인 게이트웨이는 캐시 게이트웨이

    • S3 OneZone-IA 의 내구성은 99.999999999 %

    • S3 개체는 키- 값 쌍 기반

    • S3 버킷에 접근하는 방법은 FTP X SFTP X REST O, 콘솔 O

    • S3는 앱 사용량이 많은 시간에 자동으로 확장되도록 설계 되어짐

    • S3는 AWS 네트워크에서 균등하게 확장되는 경향이 있음

    • 멀티 파트 업로드는 100MB보다 큰 파일일 경우 사용 됨

    • 미리 서명 된 URL은 URL 작성자가 제공한 특정 개체와 관련된 권한이 있음

    • 사전 서명 된 URL을 사용하면 AWS 자격 증명 없이도 개인 S3 버킷에 엑세스 가능

    • S3에서 PUT에 대한 일관성은 읽기 쓰기 일관성 후

    • S3의 초당 PUT은 3500개

    • S3의 버킷 엑세스 포인트는 가상 호스트 스타일, 경로 스타일

    • S3에서 지원하는 일관성 모델은 쓰기 일관성 후 읽기, 최종 일관성

    • S3 객체가 실수로 삭제되지 않도록 하는 권한 MFA ( Multi )

    • MFA delete의 HTTP 헤더는 x-amz-mfa

    • MFA 삭제는 루트계정만 가능

    • 객체 메타 데이터를 추가하는 메커니즘은 태그

    • S3 버킷에서 버전 관리가 된 후에는 해제가 되지 않음

    • CloudFront가 호출한 정보 소스는 오리진 서버

    • 엣지 로케이션 > 가용영역 > 리전

    • TTL ( Time to LIve )는 캐시되는 시간

    • 엣지 로케이션의 캐시되는 시간은 최대 24시간

    • 별도의 리전에서 생성하는 것을 내결함성

    • S3, EC2는 엑셀레이터가 있다

    • S3 Transfer Acceleration의 이점은 지리적으로 가장 멀리 있는 사용자

    • Snowball은 어떤 언어도 지원하지 않음 ( 전송만 수행 )

    • Elastic Beanstalk가 지원하는 것은 도커, Node.js

    • 파일의 다양한 형식으로 변환하는 것은 Elastic Transcoder의 기능

    • 여러 데이터 소스에서 비즈니스 인텔리전스를 수집하는 데 이상적인 AWS는 QuickSight

    • 환경의 특정 이벤트를 기반으로 경고 및 경보를 전송하는 AWS 서비스는 SNS

    • AWS Cognito 사인온 뿐만이 아닌 웹 애플리케이션 사용자 가입, 로그인 및 엑세스 제어를 수행 가능 facebook, google과 같은 자격 증명 공급자 역할 가능

    • EMR ( Elastic MapReduce ) 대용량 데이터 세트의 데이터 처리 및 분석을 제공

    • Lex는 음성 인식 및 대화 봇을 구축하기 위한 Amazon 서비스

    • EMR은 빅데이터 처리에 유리

    • Kinesis는 실시간 분석에 유리

    • QuickSight는 응용프로그램을 시각적으로 관측이 가능

    • 아테나는 대화형 분석시스템으로, 대화식 측면을 사용하고 데이터를 분석하며 표준 SQL 쿼리를 사용

    • DynamoDB는 푸시 버튼 스케일링 제공 및 인스턴스 크기는 런타임에의해 선택, SSD 스토리지 사용 및 3개 이상의 지역에 분산되어 있음, 일관된 읽기를 제공

    • SWF 작업은 한 번만 할당됨

    • Amazon SES는 Simple Email Service로 AWS 애플리케이션 및 서비스의 이메일을 주고 받는 데 사용 됨

    • SNS는 아마존 서비스 별로 표현 되어짐

    • SWF에 작업 과정 모음을 워크 플로라 하며 도메인이 여기 속함

    • SQS 대기열은 선택 가능 기본적으로 FIFO로 보내려고 시도만 할 뿐임 FIFO는 아님

    • NACL은 상태 비 저장 규칙이며 인바운드 및 아웃 바운드에 대해 존재 필, 보안 그룹은 상태 저장이 가능

    • 인스턴스의 모든 메타 데이터는 [IP]/latest/meta-data에 존재

    • S3 실수로 삭제하게 된 경우 버전 관리 및 MFA 삭제를 활성화 하면 됨

    • 리전당 최대 인스턴스의 수는 20개

    • 스케쥴링 쿨 타운 타이머 수정시 임계 값 변동 가능

    • CloudWatch는 디스크 읽기 작업,CPU 사용량 및 인바운드 네트워크 트래픽을 제공 ( 기본적으로 메모리 사용량은 제공 X )

    • 공개 인터넷은 0.0.0.0/0을 통해 주소가 지정

    • RDS 인스턴스에서 보조 RDS 인스턴스로 데이터를 복제할 때는 1차에서 2차 데이터 복제에 대한 요금이 없음

    • RDS 복제본은 Aurora는 지원하지 않음

    • 읽기전용 복제본은 최대 5개

    • 읽기 전용은 성능과 확장성, AZ는 재해 복구 및 내 결함성

    • EC2는 공식적으로 28개의 첨부파일을 가질 수 있음 즉 28 -1 = 27

    • 마케팅 캠페인으로 인해 앱에 대한 트래픽이 증가할 경우, ElastiCache를 사용하여 캐시 및 복제본 구성을 설정하여 트래픽을 분산

    • 클러스터 배치 그룹은 단일 가용 영역 내에 존재하는 인스턴스로 구성되어야 하며, 이러 인해 네트워크 활동의 처리량이 증가

    • 확산 배치 그룹은 여러 가용 영역에 최대 7개의 인스턴스를 가질 수 있으며, 네트워크 대기 시간을 줄임

    • 프로비저닝된 IOPS의 볼륨은 16TB의 스토리지를 허용

    • SSD 지원 볼륨은 I/O 크기 읽기 및 쓰기 작업이 있는 트랜잭션 워크로드에 최적합

    • Elastic Load Balancing의 옵션은 클래식 로드 밸런서, 응용 프로그램 부하 분산, 가중로드 밸런서가 있음.
      ELB의 옵션에는 ELB와 ALB가 있음

    • ALB는 TCP의 7레벨에서 작동

    • 네트워크 로드 밸런서는 TCP의 레벨 4에서 작동

    • 클래식 로드 밸런서는 TCP의 레벨1, 4에서 작동

    • SSL 한개로 여러 개 돌리면 ALB, SSl 종료는 ALB, ELB모드 지원

    • route 53의 기본 도메인은 50개, 요청에 따라 증감 가능

    • route 53은 존 에이펙스 레코드 ( 네이 키드 )를 지원함

    • ElastiCache는 캐싱을 위해 memcached와 redis의 두 가지 엔진을 사용

    • ElastiCache는 메모리 데이터 저장소, 샤딩 애플리케이션 요구를 위한 매커니즘을 의미

    • CloudFront 배포와 상호 작용하는 데는 CloudFormation, AWS CLI, AWS REST API 및 AWS가 제공하는 다양한 SDK를 통해 상호 작용이 가능

    • CloudFRONT의 전송비용은 무료

    • Edge는 기본적으로 24시간마다 업데이트 된 내용을 학인하지만, 값의 변경이 가능

    • CloudFront는 0초의 만료 기간을 갖도록 할 수 있으며, 캐싱이 발생하지 않음을 의미 ( 모든 콘텐츠가 만료 되어짐 )

    • 무효화 API를 사용하여 CloudFront를 즉시 객체 제거 가능

    • 일반적으로 CLacier의 데이터 검색은 3~ 5시간이 걸림

    • S3와 S3-IA의 차이는 엑세스 빈도를 의미 검색의 속도는 동일
